Obsah

git-flow je soubor git rozšíření, které poskytují vysoko úrovňové operace v repozitáři pro Vincentův Driessnův model větvení. více informací

★ ★ ★

Tento tahák ukazuje zakladní použití a výsledky git-flow příkazů

★ ★ ★

Základní tipy

  • Git flow poskytuje excelentní napovědu v příkazové řádce a výstup. Pozorně si je čtěte aby jste zjistili co se děje...
  • OSX/Windows klient Sourcetree je výborný git gui nástroj a poskytuje podporu pro git-flow
  • Git-flow je řešení založené na začlenění (merge). A nepoužívá přeskládání(rebaseing) pro funkcionalitní(feature) větve.

★ ★ ★

Instalace

  • Pro instalaci potřebujete fungujicí git instalaci.
  • Git flow funguje na OSX, Linuxu a Windows

★ ★ ★

OSX

Homebrew
$ brew install git-flow
Macports
$ port install git-flow

Linux

$ apt-get install git-flow

Windows (Cygwin)

$ wget -q -O - --no-check-certificate https://github.com/nvie/gitflow/raw/develop/contrib/gitflow-installer.sh | bash

Potřebujete wget a util-linux pro instalaci git-flow.

Pro detailní instrukce k instalaci prosím navštivte git flow wiki.

Začínáme

Git flow musí být nainicializovánaby mohl přizpůsobit vaše nastavení.

★ ★ ★

Inicializace

Začněte používat git-flow inicializací uvnitř existujicího git repozitáře:

git flow init

Budete muset odpovědět pár otázek ohledně pojmenovávacích konvencí vašich větví.
Je doporučeno použít standartní hodnoty.

Funkce

  • Vyvíjejte nové funkce pro nadcházející vydání
  • Většinou existují pouze ve vývojářských repozitářích

★ ★ ★

Vytvoření funkce

Vývoj nové funkce začíná z 'develop' větve.

Začněte vývoj nové funkce s

git flow feature start MYFEATURE

Tento příkaz vytvoří novou funkcionalitní(feature) větev založenou na 'develop' a přepne na ni

Dokončení funkce

Dokonči vývoj funkce. Tento příkaz vykoná následujicí

  • Začlení MYFEATURE do 'develop'
  • Odstraní funkcionalitní větev
  • Přepne zpět na 'develop' větev
git flow feature finish MYFEATURE

Zveřejnění funkce

Vyvíjíte funkci dohromady?
Zveřejni funkci na vzdálený server, aby ji mohli použít i ostatní uživatelé.

git flow feature publish MYFEATURE

Získaní funkce

Získejte funkci zveřejněnou jiným uživatelem.

git flow feature pull origin MYFEATURE

Můžete sledovat(track) funkci na serveru za použití git flow feature track MYFEATURE

Tvorba vydání

  • Přípravy na nové produkční vydání.
  • Umožňuje opravu minor bugů a přípravy meta-dat na vydání.

★ ★ ★

Vytvoření vydání

Pro vytvoření vydání použijte git flow release příkaz. Tento příkaz vytvoří vydávací(release) větev z 'develop' větve.

git flow release start RELEASE [BASE]

Můžete volitelně přidat [BASE] sha-1 hash revize ze které chcete začínat. Tato revize musí být v 'develop' větvy.

★ ★ ★

Je moudré zveřejnit vydávací(release) větev po jejím vytvoření, aby mohli přispívat i ostatní vývojáři. Udělejte to podobně jako když zveřejňujete funkci příkazem:

git flow release publish RELEASE

(Můžete sledovat(track) vzdálené vydání za použití
git flow release track RELEASE příkazu)

Dokončení vydání

Dokončení vydání je jeden z velkých kroků v git větvení. A musí se vykonat několik akcí:

  • Začlenění vydávací(release) větve zpátky do 'master'
  • Označkovat(Tag) vydání jeho jménem.
  • Zpět začlenit vydání do 'develop'
  • Odstranění vydávací(release) větve
git flow release finish RELEASE

Nezapomeňte odeslat(push) známky(tags) na server sgit push --tags

Hotfixy

  • Hotfixy vznikají z nutnosti reagovat okamžitě na nezamýšlený stav produkční verze.
  • Mohou vycházet z odpovídající značky(tag) na master větvi, která označuje produkční verzi.

★ ★ ★

git flow hotfix start

Jako ostatní git flow příkazy, hotfix se začíná pomocí

git flow hotfix start VERSION [BASENAME]

Parametr version tady označuje jméno nového hotfixového vydání. Volitelně můžete uvést basename vydaní ze kterého chcete začínat.

Dokončení hotfixu

Dokončením se hotfix začlení(merge) zpátky do develop a master. Dodatečně se master začlenění(merge) označí hotfixovým version.

git flow hotfix finish VERSION

Dodatek

★ ★ ★

  • Ne všechny dostupné příkazy jsou zde zahrnuty, pouze ty nejdůležitější

  • Pokud by jste chtěli přidat překlad rád je přidám.
